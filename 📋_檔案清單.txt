================================================================================
📝 檔案內容驗證
================================================================================

【build-windows.yml 驗證】

應該包含以下內容：
- name: Build Windows Executable
- on: [push, workflow_dispatch]
- runs-on: windows-latest
- 步驟：檢出代碼、設置 Node.js、安裝依賴、構建應用

【main.ts 驗證】

應該包含以下內容：
- import { app, BrowserWindow } from 'electron'
- 創建應用窗口的代碼
- 加載前端代碼的代碼
- IPC 通信的代碼

【preload.ts 驗證】

應該包含以下內容：
- import { contextBridge, ipcRenderer } from 'electron'
- contextBridge.exposeInMainWorld 的代碼
- IPC 通信的代碼

【tsconfig.electron.json 驗證】

應該包含以下內容：
- "target": "ES2020"
- "module": "commonjs"
- "outDir": "./electron/dist"
- "rootDir": "./electron"

================================================================================
⚠️ 常見錯誤
================================================================================

❌ 錯誤 1：檔案放在錯誤的位置
症狀：GitHub Actions 工作流不運行
解決：確認 build-windows.yml 在 .github/workflows/ 目錄中

❌ 錯誤 2：缺少 tsconfig.electron.json
症狀：TypeScript 編譯失敗
解決：確認 tsconfig.electron.json 在項目根目錄

❌ 錯誤 3：缺少 main.ts 或 preload.ts
症狀：Electron 編譯失敗
解決：確認這兩個檔案在 electron/ 目錄中

❌ 錯誤 4：package.json 配置不正確
症狀：找不到 build:electron 命令
解決：參考 package.json.reference 修正配置

❌ 錯誤 5：檔案名稱拼寫錯誤
症狀：各種編譯錯誤
解決：確認所有檔案名稱都正確（區分大小寫）
